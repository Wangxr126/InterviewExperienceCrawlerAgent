# 架构问题分析与重构方案

## 🚨 当前架构问题

你说得完全对！当前架构存在**职责混乱**的问题：

### 问题1：MetaExtractor vs QuestionExtractor 职责重叠

**MetaExtractor（在Architect Tools中）：**
- 位置：`backend/tools/architect_tools.py`
- 职责：从面经原文中提取元信息（公司、岗位、业务线、难度）
- 输入：面经原文
- 输出：元信息（JSON）

**QuestionExtractor（独立服务）：**
- 位置：`backend/services/crawler/question_extractor.py`
- 职责：从面经原文中提取题目
- 输入：面经原文
- 输出：题目列表（JSON）

**问题：**
- ❌ 两个都从"面经原文"提取信息
- ❌ 职责边界不清晰
- ❌ MetaExtractor应该是Architect的工具，但它在做提取工作

### 问题2：Architect Agent 的职责不清

**当前Architect Agent的工具：**
1. `MetaExtractor` - 元信息提取（❌ 这是提取工作！）
2. `KnowledgeStructurer` - 结构化解析
3. `DuplicateChecker` - 语义查重
4. `BaseManager` - 双写入库

**问题：**
- ❌ Architect既做"提取"又做"管理"
- ❌ 职责不单一

## ✅ 正确的架构应该是什么？

### 方案A：三个Agent（推荐）

```
1. Extractor Agent（提取器）
   职责：从原文提取所有信息
   - 提取元信息（公司、岗位、难度）
   - 提取题目列表
   输入：面经原文
   输出：完整的结构化数据（元信息 + 题目列表）

2. Architect Agent（架构师）
   职责：知识管理
   - 结构化解析
   - 语义查重
   - 构建知识图谱
   输入：结构化数据（来自Extractor）
   输出：知识图谱 + 去重后的数据

3. Interviewer Agent（面试官）
   职责：面试对话
   输入：用户问题
   输出：面试题目、解答、评估
```

### 方案B：两个Agent + 一个服务（当前改进）

```
1. Extractor Service（提取服务）
   职责：从原文提取所有信息
   - 提取元信息
   - 提取题目列表
   性质：服务层工具（不是Agent）
   输入：面经原文
   输出：完整的结构化数据

2. Architect Agent（架构师）
   职责：知识管理（纯粹的管理，不做提取）
   - 结构化解析
   - 语义查重
   - 构建知识图谱
   - 双写入库
   输入：结构化数据（来自Extractor）
   输出：知识图谱 + 数据库

3. Interviewer Agent（面试官）
   职责：面试对话
   输入：用户问题
   输出：面试题目、解答、评估
```

## 🔧 重构方案（推荐方案B）

### 第1步：合并提取逻辑

**创建统一的Extractor Service：**

```python
# backend/services/extractor_service.py

class ExtractorService:
    """
    统一的提取服务：从面经原文中提取所有信息
    - 元信息（公司、岗位、业务线、难度）
    - 题目列表
    """
    
    def extract_all(self, raw_text: str, source_platform: str) -> dict:
        """
        从原文中提取所有信息
        
        返回：
        {
            "meta": {
                "company": "字节跳动",
                "position": "后端研发",
                "business_line": "抖音",
                "difficulty": "medium"
            },
            "questions": [
                {
                    "question_text": "...",
                    "answer_text": "...",
                    "difficulty": "medium",
                    "question_type": "缓存题",
                    "topic_tags": ["Redis"]
                }
            ]
        }
        """
        # 1. 提取元信息（使用规则 + LLM）
        meta = self._extract_meta(raw_text, source_platform)
        
        # 2. 提取题目（使用LLM）
        questions = self._extract_questions(raw_text)
        
        return {
            "meta": meta,
            "questions": questions
        }
```

### 第2步：简化Architect Agent

**移除MetaExtractor，只保留管理工具：**

```python
# backend/agents/architect_agent.py

class KnowledgeArchitectAgent(ReActAgent):
    """
    知识架构师：纯粹的知识管理
    输入：已提取的结构化数据
    输出：知识图谱 + 数据库
    """
    def __init__(self):
        registry = ToolRegistry()
        # 只保留管理工具，移除提取工具
        registry.register_tool(KnowledgeStructurer())  # 结构化解析
        registry.register_tool(DuplicateChecker())     # 语义查重
        registry.register_tool(BaseManager())          # 双写入库
```

### 第3步：更新工作流程

```python
# backend/services/scheduler.py

def process_post(post_data):
    """处理单个帖子"""
    
    # 1. 提取所有信息（统一的Extractor Service）
    extractor = ExtractorService()
    extracted_data = extractor.extract_all(
        raw_text=post_data['content'],
        source_platform=post_data['platform']
    )
    
    # 2. 知识管理（Architect Agent）
    architect = KnowledgeArchitectAgent()
    result = architect.run({
        "meta": extracted_data["meta"],
        "questions": extracted_data["questions"]
    })
    
    return result
```

## 📊 重构前后对比

### 重构前（当前架构）

```
爬虫抓取原文
   ↓
QuestionExtractor（提取题目）
   ↓
Architect Agent
   ├─ MetaExtractor（提取元信息）❌ 重复提取
   ├─ KnowledgeStructurer（结构化）
   ├─ DuplicateChecker（查重）
   └─ BaseManager（入库）
```

**问题：**
- ❌ 提取逻辑分散（QuestionExtractor + MetaExtractor）
- ❌ Architect既做提取又做管理
- ❌ 职责不清晰

### 重构后（推荐架构）

```
爬虫抓取原文
   ↓
Extractor Service（统一提取）
   ├─ 提取元信息
   └─ 提取题目
   ↓
Architect Agent（纯粹管理）
   ├─ KnowledgeStructurer（结构化）
   ├─ DuplicateChecker（查重）
   └─ BaseManager（入库）
```

**优点：**
- ✅ 提取逻辑统一（Extractor Service）
- ✅ Architect只做管理
- ✅ 职责清晰

## 🎉 总结

### 你的问题

1. ❓ "这两个是否重复？"
   - ✅ **是的，有重复！** MetaExtractor和QuestionExtractor都在做提取工作

2. ❓ "是不是应该有三个agent？"
   - ✅ **可以！** 方案A：Extractor Agent + Architect Agent + Interviewer Agent

3. ❓ "当前提取器干的就是EXTRACTOR做的活？"
   - ✅ **是的！** MetaExtractor应该合并到Extractor中

4. ❓ "是不是应该重构这部分逻辑？"
   - ✅ **必须重构！** 当前架构职责混乱

### 推荐方案

**方案B：两个Agent + 一个服务**

```
Extractor Service（提取服务）
   ↓
Architect Agent（知识管理）
   ↓
Interviewer Agent（面试对话）
```

**优点：**
- ✅ 职责清晰
