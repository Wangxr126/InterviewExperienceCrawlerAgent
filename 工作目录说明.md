# 工作目录说明

## 目录结构

### 1. opn 工作目录（临时）
```
C:\Users\Wangxr\.cursor\worktrees\wxr_agent\opn
```
- 这是 Git worktree 创建的临时工作分支
- AI 在这里进行所有修改
- 修改完成后需要同步到实际项目目录

### 2. 实际项目目录
```
E:\Agent\AgentProject\wxr_agent
```
- 这是你的真实项目目录
- 运行后端、前端构建都在这里
- 所有修改最终要同步到这里

## 为什么有两个目录？

Cursor 使用 Git worktree 机制：
- 保护主项目不被意外修改
- 允许 AI 在隔离环境中工作
- 修改完成后手动同步

## 如何同步？

### 方法1：自动同步脚本（推荐）

运行同步脚本：
```bash
cd E:\Agent\AgentProject\wxr_agent
python sync_to_project.py
```

这会自动将以下文件从 opn 同步到实际项目：
- backend/services/crawler/nowcoder_crawler.py
- backend/services/crawler/question_extractor.py
- backend/services/finetune_service.py
- backend/services/sqlite_service.py
- backend/main.py
- web/src/views/FinetuneView.vue
- web/src/components/QuestionDialog.vue
- test4.py
- test_nowcoder_full.py

### 方法2：手动复制

```powershell
# 复制后端文件
Copy-Item "C:\Users\Wangxr\.cursor\worktrees\wxr_agent\opn\backend\services\crawler\nowcoder_crawler.py" "E:\Agent\AgentProject\wxr_agent\backend\services\crawler\nowcoder_crawler.py" -Force

# 复制前端文件
Copy-Item "C:\Users\Wangxr\.cursor\worktrees\wxr_agent\opn\web\src\views\FinetuneView.vue" "E:\Agent\AgentProject\wxr_agent\web\src\views\FinetuneView.vue" -Force
```

## 工作流程

1. **AI 修改代码** → 在 opn 目录
2. **运行同步脚本** → 复制到 E 盘
3. **构建前端**（如果修改了前端）：
   ```bash
   cd E:\Agent\AgentProject\wxr_agent\web
   npm run build
   ```
4. **重启后端**：
   ```bash
   cd E:\Agent\AgentProject\wxr_agent
   python run.py
   ```

- ⚠️ AI 的修改需要同步后才能生效
