# 递归重试机制实现完成报告

## 🎯 需求

用户反馈：重复点击【重试失败项】可以提取出内容，希望系统自动递归重试失败的任务，直到所有链接都抓取完content或达到最大重试次数。

---

## ✅ 实现方案

### 核心逻辑

在爬取和提取过程中，自动递归重试失败的任务：
1. 记录失败的URL列表
2. 循环重试直到全部成功或达到最大次数
3. 每轮重试后统计成功率
4. 如果某轮无任何成功，提前退出

### 2. 修改scheduler.py的_process_pending_tasks函数

**修改位置：** `backend/services/scheduler.py` 第289-410行

**旧逻辑（Step 3）：**
```python
# ── Step 3: 重试失败的任务 ──────────────────────────────
retry_count = _retry_failed_tasks()
if retry_count > 0:
    logger.info(f"🔄 重试成功 {retry_count} 个任务")

return processed
```

**新逻辑（递归重试）：**
```python
# ── Step 3: 递归重试失败的任务（直到全部成功或达到最大次数）──────────────────────────────
from backend.config.config import settings
max_recursive_retries = settings.crawler_recursive_retry_max

retry_round = 0
while retry_round < max_recursive_retries:
    # 检查是否还有error状态的任务
    error_tasks = sqlite_service.get_tasks_by_status("error", limit=batch_size)
    if not error_tasks:
        logger.info(f"✅ 所有任务处理完成，无需重试")
        break
    
    retry_round += 1
    logger.info(f"🔄 第 {retry_round}/{max_recursive_retries} 轮递归重试，发现 {len(error_tasks)} 个失败任务...")
    
    # 重试这些失败任务
    retry_success = 0
    for task in error_tasks:
        task_id = task["task_id"]
        url = task["source_url"]
        title = task["post_title"] or "无标题"
        
        try:
            # 1. 重新抓取详情页
            crawler = NowcoderCrawler(cookie=cfg.NOWCODER_COOKIE)
            content, image_urls = crawler.fetch_post_content_full(url)
            
            if not content or len(content) < 100:
                continue
            
            # 2. 下载图片
            image_paths = download_images(image_urls, task_id) if image_urls else []
            
            # 3. 更新为fetched状态
            sqlite_service.update_task_status(task_id, "fetched", raw_content=content, image_paths=image_paths)
            
            # 4. 立即尝试提取
            questions, status = extract_questions_from_post(...)
            
            if questions:
                # 提取成功
                count = _save_questions(questions)
                sqlite_service.update_task_status(task_id, "done", questions_count=count)
                logger.info(f"    ✅ 重试成功: {title[:40]} | 提取 {count} 道题目")
                retry_success += 1
                processed += count
            else:
                # 5. 尝试OCR
                if image_paths:
                    ocr_text = ocr_images_to_text(image_paths, task_id)
                    if ocr_text:
                        questions2, status2 = extract_questions_from_post(content_with_ocr, ...)
                        if questions2:
                            # OCR提取成功
                            count = _save_questions(questions2)
                            sqlite_service.update_task_status(task_id, "done", questions_count=count)
                            logger.info(f"    ✅ 重试成功(OCR): {title[:40]} | 提取 {count} 道题目")
                            retry_success += 1
                            processed += count
                            continue
                
                # 仍然失败
                sqlite_service.update_task_status(task_id, "error", error_msg=f"第{retry_round}轮重试仍无法提取题目")
                
        except Exception as e:
            logger.error(f"    ❌ 重试异常: {title[:40]} | {e}")
            sqlite_service.update_task_status(task_id, "error", error_msg=f"第{retry_round}轮重试异常: {str(e)[:150]}")
    
    logger.info(f"🔄 第 {retry_round} 轮重试完成：成功 {retry_success}/{len(error_tasks)} 个任务")
    
    # 如果本轮没有任何成功，提前退出
    if retry_success == 0:
        logger.warning(f"⚠️ 第 {retry_round} 轮重试无任何成功，停止递归重试")
        break

# 最终统计
final_error_count = len(sqlite_service.get_tasks_by_status("error", limit=1000))
if final_error_count > 0:
    logger.warning(f"⚠️ 递归重试结束，仍有 {final_error_count} 个任务失败")
else:
    logger.info(f"✅ 所有任务处理完成！")

return processed
```

## 🎯 工作流程

### 原有流程
```
1. pending → 抓取详情 → fetched
2. fetched → LLM提取 → done/error
3. 调用_retry_failed_tasks()重试一次
4. 结束
```

### 新流程
```
1. pending → 抓取详情 → fetched
2. fetched → LLM提取 → done/error
3. 递归重试循环（最多10次）：
   a. 查询所有error状态的任务
   b. 如果没有error任务，退出循环
   c. 对每个error任务：
      - 重新抓取详情页
      - 立即尝试LLM提取
      - 如果失败，尝试OCR
      - 更新状态（done或error）
   d. 统计本轮成功率
   e. 如果本轮无任何成功，提前退出
4. 输出最终统计
```
