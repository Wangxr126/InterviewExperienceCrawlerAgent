# 面经题目提取模型微调方案

> 目标：让小模型（qwen3:4b / gemma3:4b）在提取面经题目时达到大模型水平
> 更新时间：2026-03

---

## 一、背景与问题

当前使用本地小模型（qwen3:4b）做面经结构化提取，存在以下问题：

| 问题 | 表现 |
|---|---|
| 换行/分号识别差 | 原文换行或用分号分隔的多道题被合并为一条，或完全漏提取 |
| 提取无意义片段 | 把「然后」「好难啊」「问了一些」等过渡语提取为题目 |
| answer_text 质量低 | 小模型生成的参考答案太简短，缺乏实质内容 |
| JSON 格式错误 | 偶发输出格式错乱，导致解析失败（parse_error） |

## 二、数据收集流程

### 2.1 原始日志（自动生成）

每次爬虫提取后自动写入：

```
微调/llm_logs/{模型名}/{来源}_{日期}.jsonl
```

每条记录格式：
```json
{"ts": "2026-03-07T14:30:00", "content": "面经原文", "llm_raw": "小模型JSON输出"}
```

- `content`：去除系统提示词和模板后的纯面经原文
- `llm_raw`：小模型的原始输出（不做任何修改）

### 2.2 标注流程（微调标注页面）

1. **导入**：将 llm_logs 文件导入 SQLite `finetune_samples` 表
2. **调用大模型**：对每条 content 调用远程大模型（qwen-max / GPT-4o 等）生成高质量结果
3. **人工审核**：检查大模型结果，按需手动修改
4. **确认标注**：点击「确认标注」保存为 final_output，记录是否手动修改
5. **导出**：导出 `微调/labeled_data.jsonl` 用于训练

### 2.3 SQLite 表结构（finetune_samples）

| 字段 | 说明 |
|---|---|
| id | 主键 |
| content | 面经原文 |
| llm_raw | 小模型原始输出 |
| assist_output | 大模型辅助生成输出 |
| final_output | 最终标注（人工确认） |
| is_modified | 是否手动修改过大模型输出（0/1） |
| status | pending / labeled |
| created_at | 日志写入时间 |
| labeled_at | 标注完成时间 |

## 三、微调数据格式

导出文件：`微调/labeled_data.jsonl`

每条格式：
```json
{
  "prompt": "面经原文内容...",
  "completion": "[{\"question_text\":\"...\",\"answer_text\":\"...\",\"difficulty\":\"medium\",\"question_type\":\"技术题\",\"topic_tags\":[\"RAG\",\"LLM\"],\"company\":\"\",\"position\":\"\"}]",
  "is_modified": true,
  "labeled_at": "2026-03-07T15:00:00"
}
```

## 四、微调方案（以 qwen3:4b 为例）

### 4.1 方法选择：LoRA / QLoRA

推荐使用 **QLoRA（4-bit 量化 + LoRA）**，资源消耗低，适合本地 GPU。

```
显存需求：约 8~12GB（4-bit 量化后的 4B 模型）
训练框架：LLaMA-Factory / Unsloth
```

### 4.2 数据准备

```bash
# 从标注系统导出
python -m backend.services.finetune_service  # 或通过 API /api/finetune/export

# 转换为 Alpaca 格式（LLaMA-Factory 通用格式）
python 微调/convert_to_alpaca.py
```

Alpaca 格式：
```json
{
  "instruction": "你是面经结构化专家，从面经原文中精准提取所有面试题，输出严格的JSON数组。\n\n[...系统提示词...]",
  "input": "面经原文...",
  "output": "[{\"question_text\":\"...\"}]"
}
```

### 4.3 训练配置（LLaMA-Factory）

```yaml
# 微调/train_config.yaml
model_name_or_path: Qwen/Qwen3-4B
finetuning_type: lora
lora_rank: 16
lora_alpha: 32
lora_target: q_proj,v_proj,k_proj,o_proj,gate_proj,up_proj,down_proj
dataset: finetune_alpaca
template: qwen
cutoff_len: 4096
max_samples: 5000
per_device_train_batch_size: 2
gradient_accumulation_steps: 4
learning_rate: 1.0e-4
num_train_epochs: 3
lr_scheduler_type: cosine
warmup_ratio: 0.1
output_dir: 微调/output/qwen3-4b-interview
fp16: true
```

### 4.4 训练启动

```bash
llamafactory-cli train 微调/train_config.yaml
```

### 4.5 效果评估

评估维度：
- **题目完整性**（召回率）：原文有 N 道题，提取了几道
- **题目准确性**（精确率）：提取的是否全是有效题目
- **JSON 格式正确率**：输出能直接解析的比例
- **answer_text 质量**：参考答案是否实质性、分条列点

评估方法：
1. 预留 10% 数据作为测试集（不参与训练）
2. 对比微调前后在测试集上的指标
3. 重点检查换行/分号分隔的题目和 answer_text 质量

## 五、数据量建议

| 数据量 | 预期效果 |
|---|---|
| < 100 条 | 轻微改善，效果不稳定 |
| 100~300 条 | 明显改善格式和过滤问题 |
| 300~1000 条 | 换行/分号识别显著提升 |
| > 1000 条 | 接近大模型提取质量 |

**优先标注规则**：
1. 小模型 parse_error 的记录（格式问题）
2. 小模型 empty 但实际有题目的记录（漏提取）
3. 包含换行/分号分隔题目的记录
