# 图片识别（OCR）方案

## 整体流程

```
爬虫发现帖子
    ↓
image_utils.download_images(image_urls, task_id)
    → 保存到: backend/data/post_images/{task_id}/0.jpg
    → 存入 DB: image_paths = ["{task_id}/0.jpg", ...]
    ↓
scheduler._process_pending_tasks()
    → 读取 image_paths（相对路径列表）
    → 调用 extract_questions_from_post(image_paths=image_paths)
    ↓
MinerAgent.run()
    → 正文有题目 → 直接 LLM 提取
    → 正文无题目但有图片 → 自动调用 ocr_images_to_text(image_paths)
    ↓
ocr_images_to_text(["TASK-XXX/0.jpg"])
    → full_path = settings.post_images_dir / "TASK-XXX/0.jpg"
             = backend/data/post_images/TASK-XXX/0.jpg  ✓
    → Qwen-VL OCR 识别 → 返回文字
```

---

## OCR 服务入口

**文件**：`backend/services/crawler/ocr_service_mcp.py`

统一函数：
```python
from backend.services.crawler.ocr_service_mcp import ocr_images_to_text

ocr_text = ocr_images_to_text(
    image_paths=["TASK_XXX/0.jpg", "TASK_XXX/1.png"],
    task_id="TASK_XXX",
)
```

- `image_paths`：数据库中存储的**相对路径**列表（相对于 `backend/data/post_images/`）
- 函数内部自动拼接为绝对路径后调用视觉模型
- 返回值：所有图片识别文字拼接的字符串，全部失败返回 `""`

## 支持的 OCR 方式

通过 `.env` 的 `OCR_METHOD` 配置切换，支持三种方式：

| 方式 | 配置值 | 说明 | 所需 Key |
|------|--------|------|----------|
| 阿里云 Qwen-VL | `qwen_vl`（**默认/推荐**）| 调用阿里百炼 API，已有 `EMBED_API_KEY` 即可复用 | `OCR_API_KEY` 或 `EMBED_API_KEY` |
| Claude Vision | `claude_vision` | Anthropic Claude 视觉模型，准确率高但按图收费 | `ANTHROPIC_API_KEY` |
| 本地 MCP | `mcp` | 调用本地 `mcp-image-extractor` 预处理后再交给视觉模型 | 同上任选其一 |

### 失败回退顺序

```
主方式 → qwen_vl → claude_vision
```

如果主方式失败，依次尝试 qwen_vl → claude_vision，最终仍失败返回空字符串。

## 配置项（.env）

```env
# OCR 方式：qwen_vl（默认）/ claude_vision / mcp
OCR_METHOD=qwen_vl

# Qwen-VL 模型（可选，默认 qwen-vl-plus）
OCR_MODEL=qwen-vl-plus

# Qwen-VL API Key（可选，留空则复用 EMBED_API_KEY）
OCR_API_KEY=

# Claude Vision 需要配置此项
ANTHROPIC_API_KEY=sk-ant-xxx
```

## 图片存储规则

| 项目 | 说明 |
|------|------|
| 存储根目录 | `backend/data/post_images/` |
| 子目录 | 按 `task_id` 隔离，如 `TASK_20260310_abc123/` |
| 文件命名 | 按下载顺序编号：`0.jpg`、`1.png` 等 |
| 数据库字段 | `crawl_tasks.image_paths`，存相对路径 JSON 数组 |
| 前端访问 | `/post-images/{task_id}/{idx}.jpg`（静态挂载） |

## 在 MinerAgent 中的触发逻辑

```python
# backend/agents/miner_agent.py（简化示意）
if not text_has_questions(content) and image_paths:
    ocr_text = ocr_images_to_text(image_paths, task_id)
    if ocr_text:
        content += "\n\n## 图片内容（OCR识别）\n" + ocr_text
        # 再次调用 LLM 从增强内容中提取题目
        items, status = extract_with_llm(content)
        extraction_source = "ocr"
```

## 常见问题

**Q: Qwen-VL 识别失败？**  
A: 检查 `EMBED_API_KEY` 是否有效，或单独配置 `OCR_API_KEY`。

**Q: 图片路径不存在？**  
A: 确认爬虫已调用 `image_utils.download_images()` 下载图片，DB 中 `image_paths` 不为空。

**Q: MCP 模式报错找不到 index.js？**  
A: 在 `mcp/mcp-image-extractor/` 目录执行 `npm run build` 编译。

**Q: 想切换到 Claude Vision？**  
A: 设置 `.env` 中 `OCR_METHOD=claude_vision` 并填写 `ANTHROPIC_API_KEY`。
