## 使用流程

### 1. 访问定时任务管理
1. 启动前端开发服务器：`cd web && npm run dev`
2. 访问应用
3. 点击左侧导航栏的 "⏰ 定时任务"

### 2. 创建新任务
1. 点击 "新建任务" 按钮
2. 填写任务信息：
   - 任务名称（必填）
   - 任务类型（必填）
   - 任务描述（可选）
3. 配置调度：
   - 选择 Cron 或间隔
   - 可使用模板快速填充
4. 设置任务参数（根据任务类型动态显示）
5. 选择是否启用
6. 点击 "保存"

### 3. 管理任务
- **编辑**: 点击任务行的 "编辑" 按钮
- **启用/禁用**: 切换状态开关
- **立即执行**: 点击 "立即执行" 按钮（需确认）
- **删除**: 点击 "删除" 按钮（需确认）

### 4. 查看任务状态
任务列表显示：
- 上次运行时间
- 下次运行时间
- 已运行次数
- 当前启用状态

## 技术实现

### 前端技术栈
- Vue 3 Composition API
- Element Plus UI 组件库
- 响应式数据绑定
- 异步 API 调用

### 后端技术栈
- FastAPI RESTful API
- APScheduler 任务调度
- SQLite 数据持久化
- Pydantic 数据验证

### 数据流
```
前端界面 → API 调用 → FastAPI 路由 → SchedulerService (数据库)
                                    ↓
                            CrawlScheduler.reload_jobs()
                                    ↓
                            APScheduler (执行任务)
```

## 优势

2. **动态生效**: 任务修改后立即生效，无需重启
3. **灵活配置**: 支持 Cron 和间隔两种方式
4. **参数化**: 每个任务独立配置参数

6. **实时反馈**: 显示任务运行状态和历史

## 注意事项

1. **时区**: 所有时间使用 Asia/Shanghai 时区
2. **并发**: 同一任务不会并发执行
3. **错过执行**: 错过的任务会在 1 小时内补执行
4. **数据持久化**: 任务配置存储在 SQLite 数据库
5. **服务重启**: 重启后自动加载数据库中的任务
