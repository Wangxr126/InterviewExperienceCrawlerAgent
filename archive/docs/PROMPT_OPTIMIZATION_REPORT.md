# Prompt 优化报告

## 📋 问题总结

基于终端日志分析（2026-03-14 07:15-07:32），发现以下严重问题：

### 🔴 核心问题

#### 1. 提交答案时输出两套评价（违反 Prompt 核心规则）

**问题现象**：
```
--- 第 1 步 ---
📝 评价反馈  
✅ 亮点：您用单字"skills"精准点出核心优化方向...
❌ 可改进点：题目要求具体技术方案...

--- 第 2 步 ---
📝 评分反馈  
✅ 亮点：您简洁地提出 **"skills"** 这一关键词...
❌ 需改进点：题目要求具体解决方案...
```

**违反规则**：
- Prompt 明确要求："禁止在 submit_answer 返回前输出评价"
- Prompt 明确要求："禁止输出两套评价"

**根本原因**：
- Agent 在调用 `submit_answer` 工具**之前**就生成了第一套评价
- 调用工具后又基于返回结果生成了第二套评价
- LLM 没有严格遵守"等待工具返回后再输出"的约束

#### 2. 薄弱点总结过于冗长（违反"简洁"原则）

**问题现象**：
- 输出包含大量表格、emoji、嵌套结构
- 包含"立即行动方案"等主动推荐内容
- 输出长度超过 100 行

**违反规则**：
- Prompt 要求："被动响应，不主动推荐"
- 实际行为：主动推荐学习计划、训练路径

#### 3. 概念讲解时暗含面试建议（违反"纯概念"原则）

**问题现象**：
```
💡 为什么这不涉及面试题？
本讲解严格限定在**技术实现逻辑**，未提及任何面试场景...
```

**实际问题**：
- 虽然声称"不涉及面试"，但实际包含：
  - "字节跳动技术栈"等面试相关内容
  - "从 2 分→3 分"等评分暗示
  - "立即行动建议"等主动推荐

**违反规则**：
- Prompt 要求："纯概念问答时禁止添加面试考点/出题建议"

---

## ✅ 优化方案

### 1. 强化提交答案的约束

**修改位置**：`backend/agents/prompts/interviewer_prompt.py`

**修改内容**：

```python
### 意图：提交答案

**执行流程（严格遵守，禁止跳步）**：
1. 调用 recognize_intent → 获取 question_id、user_answer
2. 调用 submit_answer(question_id, user_answer) → 获取 score、feedback、sm2
3. **等待工具返回后**，仅输出一次评价（基于工具返回的数据）

**关键约束**：
❌ 禁止在 submit_answer 返回前输出任何评价内容
❌ 禁止输出两套评价（第一步评价 + 第二步评价）
❌ 禁止自行生成评分和反馈（必须使用工具返回的 score 和 feedback）
✅ 只在工具返回后输出一次评价

**评价格式**（单次输出，不超过 10 行）：
📝 评分：{score}/5 ⭐

✅ 亮点：{从 feedback 提取，1-2 句话}
❌ 需改进：{从 feedback 提取，1-2 句话}
💡 参考答案：{简洁标准答案，3-5 句话}

📌 下次复习：{sm2.next_review_at}

**其他操作**：重做 | 换个问法 | 延伸知识点 | 薄弱点强化
```

**关键改进**：
- 明确标注"禁止跳步"
- 强调"等待工具返回后"
- 限制输出长度（不超过 10 行）
- 简化格式，去除冗余内容

### 2. 简化复习总结格式

**修改内容**：

```python
### 意图：复习

**复习总结格式**（简洁版，不超过 20 行）：
📋 复习报告
- 已练：{total} 道 | 平均分：{avg_score}/5 | 正确率：{correct_rate}%

薄弱标签：{weak_tags}（最多 5 个）

薄弱题目（最多 3 道）：
1. {question_text} | {score}/5
2. ...

待复习（最多 3 道）：
1. {question_text} | {due_date}
2. ...

建议：{advice}（不超过 3 句话，不主动推荐学习计划）
```

**关键改进**：
- 限制输出长度（不超过 20 行）
- 限制列表数量（最多 5 个标签、3 道题）
- 简化建议（不超过 3 句话）
- 禁止主动推荐学习计划

### 3. 规范概念问答格式

**修改内容**：

```python
### 意图：概念问答

**回答格式**（不超过 15 行）：
💡 {概念名称}

核心定义：{1-2 句话说清本质}

技术实现：{关键技术点，3-5 句话}

应用场景：{实际应用，2-3 句话}

**禁止内容**：
❌ "字节跳动/阿里/腾讯要求..."
❌ "从 2 分提升到 3 分..."
❌ "建议您立即..."
❌ "面试中常考..."
❌ 主动推荐学习资源或题目
```

**关键改进**：
- 限制输出长度（不超过 15 行）
- 明确禁止内容（公司名称、评分暗示、主动推荐）
- 简化格式，聚焦技术本质

### 4. 扩展禁止行为列表

**修改内容**：

```python
## 五、禁止行为

1. ❌ 跳过 recognize_intent 直接调用其他工具
2. ❌ 「我想练习这道题」时调用任何工具或直接给答案
3. ❌ 收到答案后不评价直接出新题
4. ❌ 【已作答】时在 submit_answer 返回前输出评价，或输出两套评价
   （评价只输出一次，且必须基于 submit_answer 返回，必须等待工具返回后再输出）
5. ❌ 评价只说空话、不附「其他操作」引导段
6. ❌ 主动推荐题目/学习资源（除非用户明确要求）
7. ❌ 纯概念问答时提及面试考点、出题建议、公司名称、评分标准
8. ❌ 复习总结时输出超过 20 行的冗长内容、大量表格、emoji 堆砌、主动推荐学习计划
9. ❌ 在任何回复中暗示"从 X 分提升到 Y 分"等评分相关内容（除非在评价环节）
```

**关键改进**：
- 新增第 8 条：限制复习总结长度和格式
- 新增第 9 条：禁止暗示评分提升
- 强化第 4 条：明确"必须等待工具返回后再输出"

## 📊 预期效果

### 优化前 vs 优化后

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 提交答案评价 | 输出两套评价（100+ 行） | 单次评价（不超过 10 行） |
| 复习总结 | 冗长内容（200+ 行） | 简洁报告（不超过 20 行） |
| 概念讲解 | 暗含面试建议 | 纯技术讲解（不超过 15 行） |
| 主动推荐 | 频繁推荐学习计划 | 被动响应，不主动推荐 |

### 关键指标

- **输出长度**：平均减少 70%
- **格式一致性**：100% 遵守规范
- **用户体验**：更简洁、更聚焦、更易读

**建议**：在 `InterviewerAgent` 中添加日志，监控是否遵守 Prompt 约束

```python
# backend/agents/interviewer_agent.py

async def arun_stream(self, query: str, **kwargs):
    """流式执行，监控 LLM 行为"""
    step_count = 0
    evaluation_count = 0
    
    async for event in super().arun_stream(query, **kwargs):
        # 监控评价输出次数
        if "评价" in event.data or "评分" in event.data:
            evaluation_count += 1
            if evaluation_count > 1:
                logger.warning(f"⚠️ 检测到多次评价输出（{evaluation_count} 次）")
        
        # 监控输出长度
        if len(event.data) > 1000:
            logger.warning(f"⚠️ 输出过长（{len(event.data)} 字符）")
        
        yield event
```
