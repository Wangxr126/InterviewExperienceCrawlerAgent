# 牛客图片提取问题修复完成 ✅

## 问题发现

用户反馈：之前下载的不是用户上传的面试题图片，而是页面图标。

## 根本原因

通过调试脚本 `调试牛客图片提取.py` 分析页面结构，发现：

**用户上传的图片存储在 `imgMoment` 数组中，不在 `content` 文本字段里！**

调试输出显示：
```
[IMG] root.prefetchData.2.ssrCommonData.contentData.imgMoment: [数组，1项]
```

## 原代码问题

`backend/services/crawler/nowcoder_crawler.py` 中的 `_collect_image_urls_from_json` 函数存在以下问题：

1. ❌ 只从 `content` 字段（Markdown文本）中提取图片URL
2. ❌ 使用正则 `r'window\.__INITIAL_STATE__\s*=\s*(\{.*?\});'` 导致JSON解析失败
3. ❌ 没有读取 `imgMoment` 数组（真正的用户上传图片存储位置）

## 修复方案

修改了 `_collect_image_urls_from_json` 函数（第477-554行）：

### 关键改进

1. ✅ **使用手动JSON解析**：避免正则匹配未闭合字符串导致的失败
2. ✅ **从 imgMoment 数组提取图片**：这是用户上传图片的真实存储位置
3. ✅ **支持两种页面类型**：
   - Feed页面：`prefetchData[X].ssrCommonData.contentData.imgMoment`
   - Discuss页面：`postDetail.imgMoment`
4. ✅ **过滤非内容图片**：排除表情包、头像、UI图标

### 核心代码逻辑

```python
# feed页面：从 imgMoment 数组提取
content_data = ssr.get('contentData')
if isinstance(content_data, dict):
    img_moment = content_data.get('imgMoment', [])
    if isinstance(img_moment, list):
        for img_item in img_moment:
            if isinstance(img_item, dict):
                src = img_item.get('src', '')
                if src and _is_user_content_image_url(src):
                    images.append(src)

# discuss页面：从 imgMoment 数组提取
post_detail = data['postDetail']

```
