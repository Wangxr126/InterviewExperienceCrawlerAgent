# 超时问题快速修复

## 🐛 问题
```
⚠️ 响应超时（90s），LLM 服务可能繁忙，请稍后重试
```

## 🎯 原因
- Agent 最大步数：100 步（需要更多时间）
- 后端超时：90 秒（太短）
- 前端超时：90 秒（太短）

## ✅ 修复内容

### 1. Agent 最大步数
**文件**：`.env`
```bash
INTERVIEWER_MAX_STEPS=100  # 从 8 改为 100
```

### 2. 后端超时
**文件**：`backend/main.py`
```python
timeout=300.0  # 从 90.0 改为 300.0（两处）
```

### 3. 前端超时
**文件**：`web/src/views/ChatView.vue`
```javascript
setTimeout(() => ctrl.abort(), 300000)  // 从 90000 改为 300000
```

## 🚀 部署

```bash
# 1. 重新构建前端
cd web
npm run build

# 2. 重启服务
cd ..
python run.py

# 3. 测试（等待 1-3 分钟）
# 打开 http://localhost:8000
# 输入"我想练习这道题：上下文压缩的问题"
```

## 📊 效果对比

| 配置 | 修改前 | 修改后 |
|------|--------|--------|
| Agent 步数 | 8 步 | 100 步 |
| 后端超时 | 90 秒 | 300 秒 |
| 前端超时 | 90 秒 | 300 秒 |
| 结果 | ❌ 超时 | ✅ 足够时间 |
