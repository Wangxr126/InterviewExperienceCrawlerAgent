# 对话功能修复测试指南

## 快速测试步骤

### 前置条件
1. 确保后端服务已启动：`python run.py`
2. 确保前端已构建：`cd web && npm run build`
3. 打开浏览器访问：`http://localhost:8000`

---

## 测试场景 1：对话框输入测试

### 目标
验证在对话框中输入消息不会产生重复请求

### 步骤
1. 打开浏览器开发者工具（F12）
2. 切换到 **Network** 标签
3. 点击左侧导航 **💬 练习对话**
4. 在输入框输入："你好"
5. 按 **Enter** 键发送

### 预期结果
✅ Network 标签中只有 **1个** `/api/chat/stream` 请求
✅ 控制台显示：
```
🟣 send() 被调用
🟣 text: 你好
🟣 loading.value: false
🟣 sendInProgress: false
```
✅ 发送按钮变为加载状态（显示 loading 图标）
✅ 快捷问题按钮全部变为禁用状态
✅ 收到回复后，按钮恢复正常

### 失败标志
❌ Network 中出现 2个或更多 `/api/chat/stream` 请求
❌ 后端日志显示多次 `[Stream ←]` 日志
❌ 按钮没有禁用

---

## 测试场景 5：后端日志验证

### 目标
验证后端不再产生重复的调试日志

### 步骤
1. 打开后端终端（运行 `python run.py` 的窗口）
2. 在前端发送一条消息："你好"
3. 观察后端日志输出

### 预期结果
✅ 只看到 **1次** `[Stream ←]` 日志
✅ **不再看到** 以下调试日志：
   - `[Stream] 开始调用 orchestrator.chat(), attempt=1`
   - `[Stream] orchestrator.chat() 返回成功`
   - `[Stream] reply 类型: ...`
   - `[Stream] thinking_steps 类型: ...`
✅ 日志简洁清晰

### 示例正确日志
```
2026-03-09 04:10:00 | INFO    | [Stream ←] user=user_001 | 你好
2026-03-09 04:10:02 | INFO    | [Stream →] 50chars, thinking=0steps
```

### 失败标志
❌ 看到多次 `[Stream ←]` 日志
❌ 看到 `attempt=1, attempt=2, attempt=3` 同时出现
❌ 看到大量调试日志

## 故障排查

### 问题 1：按钮一直禁用
**可能原因**：`sendInProgress` 标志未正确重置
**解决方法**：
1. 打开浏览器控制台
2. 输入：`window.location.reload()`
3. 如果问题持续，检查是否有 JavaScript 错误

### 问题 2：仍然发送多个请求
**可能原因**：浏览器缓存了旧版本代码
**解决方法**：
1. 按 Ctrl+Shift+R 强制刷新
2. 或清除浏览器缓存
3. 或使用无痕模式测试

### 问题 3：控制台报错
**可能原因**：代码修改有误
**解决方法**：
1. 查看具体错误信息
2. 检查是否所有文件都已正确修改
3. 重新构建前端：`cd web && npm run build`
