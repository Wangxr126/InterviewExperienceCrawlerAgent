## 问题1：微调标注页面自动导入日志 ✅

**问题描述**：
- 每次打开微调标注页面时，会自动导入所有日志
- 导致大量失败记录（'str' object has no attribute 'isoformat'）
- 用户希望手动控制导入时机

**修复方案**：
移除`FinetuneView.vue`中`onMounted`的自动导入调用

**修改文件**：
- `web/src/views/FinetuneView.vue`

**修改内容**：
```javascript
// 修改前
onMounted(async () => {
  await loadStats()
  await loadLogFiles()
  await autoImportAll()  // ❌ 自动导入
})

// 修改后
onMounted(async () => {
  await loadStats()
  await loadLogFiles()
  // 不再自动导入，用户需要手动点击按钮导入 ✅
})
```

## 问题2：题库浏览页面不刷新数据 ✅

**问题描述**：
- 在数据采集页面删除所有数据
- 切换到题库浏览页面，仍显示旧数据
- 必须手动刷新浏览器才能看到新数据

**根本原因**：
- `App.vue`使用`v-show`而不是`v-if`
- 没有使用`<keep-alive>`包裹组件
- `onActivated`钩子不会被触发（只在`<keep-alive>`中有效）

**修复方案**：
1. 在`App.vue`中为`BrowseView`传递`isActive`属性
2. 在`BrowseView.vue`中监听`isActive`变化
3. 当从非激活变为激活时，重新加载数据

**修改文件**：
- `web/src/App.vue`
- `web/src/views/BrowseView.vue`

**修改内容**：

**App.vue**：
```vue
<!-- 修改前 -->
<BrowseView v-show="currentView === 'browse'" :meta="meta"
            @send-to-chat="onSendToChat" />

<!-- 修改后 -->
<BrowseView v-show="currentView === 'browse'" :meta="meta"
            :is-active="currentView === 'browse'"
            @send-to-chat="onSendToChat" />
```

**BrowseView.vue**：
```javascript
// 1. 添加 isActive prop
const props = defineProps({
  meta: { type: Object, default: () => ({}) },
  isActive: { type: Boolean, default: false }  // ✅ 新增
})

// 2. 用 watch 替换 onActivated
// 修改前
onMounted(loadQuestions)
onActivated(loadQuestions)  // ❌ 不会触发

// 修改后
onMounted(loadQuestions)
watch(() => props.isActive, (newVal, oldVal) => {
  if (newVal && !oldVal) {
    // 从非激活变为激活，重新加载数据 ✅
    loadQuestions()
  }
})
```

## 技术要点

### Vue组件生命周期钩子

| 钩子 | 触发时机 | 使用场景 |
|------|---------|---------|
| `onMounted` | 组件挂载后 | 初始化数据 |
| `onActivated` | 组件激活时 | **仅在`<keep-alive>`中有效** |
| `watch` | 响应式数据变化 | 监听prop变化 |

### v-show vs v-if

| 指令 | 行为 | 生命周期 | 适用场景 |
|------|------|---------|---------|
| `v-if` | 条件渲染 | 每次切换都会销毁/重建 | 不频繁切换 |
| `v-show` | 切换display | 只挂载一次 | 频繁切换 |

**本项目使用`v-show`**：
- 优点：切换快，保持状态
- 缺点：`onActivated`不触发
- 解决：用`watch`监听`isActive` prop
