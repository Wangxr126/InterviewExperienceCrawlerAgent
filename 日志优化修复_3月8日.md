# 日志优化修复 - 3月8日

## 修改内容

### 1. 注释掉"正在抓取"的日志 ✅

**问题**：
- 每抓取一个帖子都会打印"📄 正在抓取: xxx"
- 当批量抓取时（如37条任务），日志过多，影响可读性

**修复**：
- 删除了`_process_pending_tasks`中的"正在抓取"日志
- 保留了批次开始日志："📥 开始抓取详情，本批 X 条 pending 任务"
- 保留了成功/失败日志："✅ 抓取详情成功" / "❌ 抓取详情失败"

**修改位置**：
- `backend/services/scheduler.py` 第163行（已删除）

**日志对比**：


## 日志级别说明

### 当前日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| INFO | 重要流程节点 | 开始抓取、抓取成功、重试完成 |
| WARNING | 需要注意的问题 | 重试失败、内容过短 |
| ERROR | 错误信息 | 抓取异常、重试异常 |
| DEBUG | 详细调试信息 | （已移除的"正在抓取"可改为此级别） |

### 日志优化原则

1. **批次级别**：打印批次开始和结束
2. **结果级别**：打印成功/失败结果
3. **避免过程日志**：不打印每个任务的处理过程
4. **保留关键信息**：标题、字数、图片数、错误信息
