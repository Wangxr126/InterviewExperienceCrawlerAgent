# MinerAgent 提取失败分析报告

> 生成时间：2026-03-11  
> 基于实际运行日志分析

---

## 一、失败类型总览

| 失败类型 | 涉及面经 | 状态码 | 根因 |
|---------|---------|--------|------|
| LLM 返回自然语言而非 JSON | 蚂蚁大模型Agent二面、拼多多后端 | `parse_error` | Prompt 格式约束不强 |
| LLM 拒绝回答 | 数字马力面经（Java岗） | `model_refused_fallback` | 叙述性面经被误判为无关内容 |
| 重试3次仍失败 | 蚂蚁大模型Agent二面 | `parse_error` × 3 | 重试未修改 Prompt，同样失败 |
| 提取不完整 | 拼多多后端（5题只提取到3题） | 部分成功 | 第3次降级调用截断了内容 |

---

## 二、详细失败原因

### 失败类型 1：`parse_error` — LLM 返回的不是纯 JSON

**涉及面经：** 蚂蚁大模型Agent算法二面、拼多多后端面经

**现象：**
```
LLM 返回内容无法解析为 JSON 数组，提取失败（已入库 status=error）
```

**LLM 实际返回（示例）：**
```
Common

以下是蚂蚁集团（Ant Group）算法岗位秋招第二轮面试中涉及的13个问题...

### 1. 你的 Agent 系统Prompt是怎么设计和迭代的？
...
```

**根本原因：**
- MinerAgent 要求 LLM 返回纯 JSON 数组，但 Prompt 的格式约束不够强硬
- LLM 在遇到"感觉需要解释"的情况时会忽略格式要求，转而输出自然语言 + Markdown
- 这段文本无法被 `json.loads()` 解析，触发 `parse_error`

**重试3次也全失败的原因：**
- 重试使用的是完全相同的 Prompt
- LLM 每次都给出类似的自然语言格式，问题没有被修复

---

### 失败类型 2：`model_refused_fallback` — LLM 拒绝回答

**涉及面经：** 数字马力面经（Java 岗，叙述性总结型）

**现象：**
```
[MinerAgent] Agent 返回拒绝文本：抱歉，我无法回答这个问题。
[MinerAgent] 执行失败，降级为直接 LLM 调用: model_refused_fallback
```

**面经内容特点：**
- 非明确编号的问答格式，而是叙述性总结
- 例如："Java基础问得挺全：ArrayList/List区别、扩容为啥是1.5倍、HashMap原理"
- 题目隐含在叙述中，不是直接列出

**根本原因：**
- MinerAgent 的任务 Prompt 过于强调"提取明确列出的题目"
- LLM 遇到叙述性内容时，认为没有明确题目可提取，触发拒绝
- `is_unrelated` 检测逻辑也可能误判叙述型面经为无关内容

---

### 失败类型 3：提取不完整

**涉及面经：** 拼多多后端面经（实际 5 题，只提取到 3 题）

**现象：**
- 前两次因 `parse_error` 失败
- 第3次重试通过 `Finish` 工具返回，但只包含了前3道题

**根本原因：**
- 第3次重试切换了策略（通过 `Finish` 工具），LLM 截断了输出内容
- 重试机制没有检测提取数量是否合理

---

## 三、修复方案

### 修复1：强化 JSON 输出约束 Prompt

在 MinerAgent 的 Prompt 末尾加入强制格式约束：

```
## 输出格式（严格遵守）
你的回复必须是且只能是一个合法的 JSON 数组，格式如下：
[{"question_id": 1, "question_text": "题目文本", "question_type": "八股"}]

禁止输出任何 JSON 之外的文字，包括：
- 解释性文字
- 说明文字  
- Markdown 标题格式
- 核对说明
不要说"以下是..."，不要说"我已核对..."，直接输出 JSON 数组。
```

### 修复2：重试时注入格式纠错指令

重试 Prompt 中追加错误提示：

```
【重要】上一次你返回的内容不是合法的 JSON 数组，解析失败。
这次请只输出 JSON 数组，不要有任何前缀、说明或 Markdown 格式。
```

### 修复3：支持叙述性面经提取

在 Prompt 中明确说明叙述性面经的处理方式：

```
面经格式可能有两种：
1. 明确编号格式："1. 问题xxx" —— 直接提取
2. 叙述性总结格式："问了xxx、yyy、zzz" —— 从叙述中归纳出具体题目，不能拒绝
无论哪种格式，都必须提取所有能识别的题目。
```

### 修复4：修复 `is_unrelated` 误判逻辑

检查 `is_unrelated` 判断条件，确保面经类内容不被误判为无关内容，即使格式是叙述型。

### 修复5：重试后校验提取数量

如果重试成功但提取数量明显少于原文题目数量（比如少于50%），应记录警告或再次重试。

## 四、优先级

| 优先级 | 修复项 | 预期效果 |
|--------|--------|----------|
| P0 | 强化 JSON 输出约束 Prompt | 解决 80%+ 的 parse_error |
| P0 | 重试时注入纠错指令 | 解决重试无效问题 |
| P1 | 支持叙述性面经提取 | 解决 model_refused 问题 |
| P1 | 修复 is_unrelated 误判 | 防止叙述型面经被过滤 |
| P2 | 重试后校验提取数量 | 防止提取不完整 |
